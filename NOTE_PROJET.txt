Non, tu n'as pas besoin d'installer le client NFS sur la machine physique Windows 11.

Voici ce que tu dois faire :

Sur la machine VM du serveur d'application (ou serveur web) :
Tu dois effectivement installer le client NFS en exécutant la commande suivante sur cette machine :


sudo apt install nfs-common

Sur ta machine physique avec Windows 11 :
Windows ne supporte pas directement NFS en natif comme Ubuntu. Pour accéder à un partage NFS depuis Windows 11, tu dois soit :

Utiliser un outil tiers qui permet de monter des partages NFS sur Windows (comme Windows Services for UNIX ou un outil NFS tiers).
Ou bien, envisager de configurer Samba à côté du NFS sur ton serveur de stockage pour permettre un accès plus facile aux machines Windows via le protocole SMB, qui est plus adapté à Windows.
Donc, en résumé :

Installe le client NFS uniquement sur les machines virtuelles (serveur d'application ou serveur web).
Pour la machine physique Windows 11, si tu veux accéder au serveur de stockage, envisage une solution alternative comme Samba ou un outil NFS tiers pour Windows.
